1. EXECUTIVE SUMMARY
--------------------------------------------------------------------------------

Nexus is a Discord-style social application for gamers. Each game is modeled as
a "hub" with text channels and voice channels. Users authenticate, join hubs,
chat in real time, manage friends and direct messages (DMs), receive
notifications, and join WebRTC voice sessions.

The product is a full-stack TypeScript web application (SSR) with an iOS shell
built with Capacitor. The iOS app does not package a standalone offline SPA as
its runtime; it loads a production HTTPS deployment of the same web app inside
a native WebView (required because server-side functions / auth / APIs are
origin-bound).

Supabase is the backend-as-a-service (auth, Postgres, Realtime, storage).
LiveKit provides real-time voice. Codemagic builds and signs the iOS IPA and
can upload to TestFlight. Cloudflare (Workers / Nitro) is the intended host
for the HTTPS web origin.

6. BACKEND — SUPABASE
--------------------------------------------------------------------------------

Role of Supabase
  Backend only: authentication, relational data, row-level security (RLS),
  Realtime subscriptions, and object storage. It does NOT host the SSR web
  application UI.

SQL layout under supabase/:
  migrations/*.sql          Canonical schema (CLI / forward-only)
  manual/01..08_*.sql       Legacy SQL Editor catch-up (prefer migrations)
  dangerous/00_reset_nexus  Guarded destructive wipe (local/empty only)
  00_reset_nexus.sql        Stub — blocks accidental wipe
  verify_schema.sql         Read-only live project inventory
  seed.sql                  Demo catalog seed

Legacy manual scripts (prefer matching migration):
  01_backfill_profiles.sql  Profile backfill
  02_phase3_friends_dms.sql Friends + DM RPCs
  03_phase4_notifications.sql Notifications triggers
  04_phase6_launch.sql     Reports + ban fields
  05_security_hardening.sql Security hardening
  06_mention_tag.sql        Mention / tag support
  07_storage.sql            Avatars + attachments buckets
  08_admin_ops.sql          Admin media columns, hub-media bucket, role bypass

Core domain tables (representative)
  profiles, games, hubs, hub_members
  text_channels, voice_channels
  messages, message_reactions
  friend_requests, friendships, blocks
  dm_threads, dm_participants, dm_messages
  notifications, user_prefs
  reports (+ profiles.banned_at / ban_reason)

Privileged operations
  Service role key is used only on the server (createServerFn). The anon key
  is the only Supabase secret exposed to the browser (VITE_*).

Realtime
  Hub chat and related social surfaces subscribe for live updates when not
  in mock mode.

Storage
  Avatars and message / DM attachments when storage SQL is applied.

7. AUTHENTICATION
--------------------------------------------------------------------------------

Implemented
  - Email + password (register / login)
  - Password reset flow (/forgot-password, /reset-password)
  - Google OAuth (Supabase provider must be enabled)
  - Discord OAuth (Supabase provider must be enabled)
  - Session via Supabase Auth; profile ensure/create on first session
  - Ban check: banned profiles are signed out with reason toast
  - Age gate: minimum 13 on registration

Deferred
  - Steam OAuth (UI shows "Soon")

Native (iOS) OAuth
  - Uses Capacitor Browser with skipBrowserRedirect
  - Deep link callback: com.lilnetro0.nexus://auth/callback
  - Must be allowlisted in Supabase Auth → URL Configuration together with
    the production HTTPS site URL

Mock / demo
  If Supabase is not configured (or VITE_USE_MOCK=1), UI runs with sample
  data and honest “demo / preview” messaging where applicable.

--------------------------------------------------------------------------------
9. VOICE — LIVEKIT DETAIL
--------------------------------------------------------------------------------

Flow
  1. Authenticated client requests a token via createVoiceToken (server fn).
  2. Server validates identity, ban state, and membership
     (hub voice channel or DM thread participation).
  3. Mint LiveKit JWT (publish/subscribe).
  4. Client joins room with livekit-client.
  5. VoiceDock mirrors mute / leave state.

Room naming
  - Hub: channel livekit_room_name or nexus-{channelId}
  - DM: nexus-dm-{threadId}

Env (server-only)
  LIVEKIT_URL, LIVEKIT_API_KEY, LIVEKIT_API_SECRET

Without LiveKit keys, the UI still offers join but in preview/stub mode
(no real WebRTC).

10. iOS / CAPACITOR / APP STORE PATH
--------------------------------------------------------------------------------

Why Capacitor (not Flutter / React Native rewrite)
  The codebase is a React / TanStack Start web app. Codemagic “React Native”
  or Flutter workflows do not apply. Capacitor wraps the product in a native
  iOS shell for TestFlight / App Store.

App identity
  Bundle ID   : com.lilnetro0.nexus
  Display name: Nexus
  App Store Connect app created (SKU example: nexus-ios-001)
  Primary locale may be Arabic depending on App Store Connect settings

Remote shell model
  The WebView loads CAPACITOR_SERVER_URL (HTTPS origin of the deployed web
  app). Without that variable at build time, TestFlight shows a placeholder
  instructing to deploy HTTPS and rebuild.

Build helpers
  scripts/prepare-capacitor-www.cjs  — builds www/ entry from Nitro public
  scripts/inject-capacitor-server-url.cjs — forces server.url into iOS config
  npm run mobile:ios / cap:sync

Native plugins
  @capacitor/app, browser, keyboard, splash-screen, status-bar

Privacy / store compliance (native)
  PrivacyInfo.xcprivacy
  ITSAppUsesNonExemptEncryption = false
  Usage strings: microphone, camera, photo library
  URL scheme for OAuth deep links

CI — Codemagic
  Workflow web-smoke     : npm ci + npm run build
  Workflow ios-capacitor : build web, Cap sync, inject URL, sign, IPA,
                           publish to TestFlight (API key integration)
  Env group example name : "test" must contain CAPACITOR_SERVER_URL
  Signing                : Apple Distribution cert + App Store profile
                           for com.lilnetro0.nexus (via App Store Connect API)

Operational notes already encountered
  - package-lock must stay in sync for npm ci
  - TestFlight "invite code" is for public links; internal testing uses email
    invite
  - External beta review needs Beta App Information / Review contact fields
  - Concurrent beta review of another build returns 422 (wait or use internal)

11. HOSTING & ENV CONFIGURATION
--------------------------------------------------------------------------------

Recommended web host
  Cloudflare Workers (Nitro Cloudflare output). Alternate: Lovable publish.
  Supabase is NOT the website host.

Critical environment variables
  VITE_SUPABASE_URL              Public
  VITE_SUPABASE_ANON_KEY         Public
  SUPABASE_SERVICE_ROLE_KEY      Server only
  VITE_USE_MOCK                  Optional force-mock
  ADMIN_USER_IDS                 Server only (admin allowlist)
  LIVEKIT_URL / API_KEY / SECRET Server only
  SITE_URL                       Production HTTPS origin
  CAPACITOR_SERVER_URL           Same HTTPS origin for iOS builds (Codemagic)

Supabase Auth URL Configuration must include
  - https://<production-host>/**
  - com.lilnetro0.nexus://auth/callback

12. SECURITY & COMPLIANCE NOTES
--------------------------------------------------------------------------------

  - CSRF middleware on Start server entry
  - Security headers in server layer
  - No service role in VITE_* variables
  - RLS expected on member-scoped tables (SQL scripts)
  - Admin actions require allowlisted user IDs on the server
  - Banned users cannot obtain voice tokens / are signed out on load
  - Age gate 13+; legal pages ship in-app
  - Encryption export answered via ITSAppUsesNonExemptEncryption=false for
    standard HTTPS use (confirm with counsel for final App Store answers)

App Store risk note
  Remote WebView shells can face review scrutiny (Guideline 4.2). Mitigate by
  shipping real native polish (mic, deep links, splash, safe areas), a live
  backend, and a clear demo of chat + voice for reviewers.
