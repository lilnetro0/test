# NEXUS DESIGN LANGUAGE (NDL)

Version 1.0

# Purpose

This document defines the permanent design philosophy for Nexus.

Every future feature, screen, interaction and visual decision must follow this document.

This document has higher priority than individual implementation prompts unless explicitly overridden.

Nexus must maintain a consistent product identity from the login screen to the deepest settings page.

---

# Product Vision

Nexus is not trying to become another Discord.

Nexus is building the premium gaming platform for Arabic-speaking gamers.

The product should feel:

- Premium
- Elegant
- Modern
- Fast
- Effortless
- Native
- Highly polished

The experience should communicate quality before features.

When users open Nexus, they should immediately feel they are using a world-class mobile application rather than a responsive website.

---



# Core Design Principles

Every design decision must satisfy all of these principles.

## 1. Simplicity

Never overwhelm the user.

Every screen should answer:

Where am I?

What can I do?

What should I do next?

If an element does not help answer those questions, remove it.

---



## 2. Premium Feel

Premium does not mean adding effects.

Premium means:

- confident spacing
- excellent typography
- smooth motion
- balanced layouts
- subtle details
- visual consistency

Luxury comes from restraint.

---



## 3. Mobile First

Design for iPhone first.

Desktop adapts from mobile.

Never build desktop layouts then compress them into a phone.

Every interaction should feel natural using one hand.

---



## 4. Native Experience

The application should never feel like a website.

Every page should resemble a native application.

Avoid common web UI patterns.

If a page feels like a responsive website, redesign it.

---



# Visual Identity

The overall appearance should feel:

Elegant before flashy.

Minimal before decorative.

Professional before playful.

Modern before trendy.

Timeless instead of fashionable.

---



# Color Philosophy

Dark theme is the primary identity.

The interface should use:

- deep dark surfaces
- soft layered backgrounds
- restrained cyan accent
- subtle gradients
- minimal glow

Avoid:

- RGB lighting
- cyberpunk aesthetics
- neon overload
- rainbow gradients
- excessive blur

Accent color should guide attention, not decorate the interface.

---



# Typography

Typography should be the primary design element.

Prefer strong hierarchy over large containers.

Use typography to communicate importance.

Large titles should be used sparingly.

Arabic typography must feel natural.

Do not apply Latin typography rules to Arabic.

Never uppercase Arabic.

Avoid excessive tracking.

---



# Layout

Whitespace is a feature.

Every screen should breathe.

Do not fill empty space simply because it exists.

Avoid giant cards covering entire screens.

Use spacing before borders.

Use typography before containers.

---



# Cards

Cards should only exist when they improve readability.

Do not wrap every component inside a card.

Cards should feel lightweight.

Prefer sections when possible.

---



# Visual Hierarchy

Every screen should have one primary focus.

The eye should naturally move:

Primary content

↓

Secondary information

↓

Actions

↓

Supporting details

Never give every component equal visual importance.

---



# Navigation

Navigation should always feel obvious.

Users should never wonder:

"Where do I go now?"

Icons must communicate meaning.

Branding should never reduce usability.

Avoid using the logo as the primary navigation element unless its purpose is immediately obvious.

Navigation must prioritize clarity over aesthetics.

---



# Discover Experience

Discover should inspire users.

It should not feel like an advanced search engine.

Reduce visible filters.

Move advanced filtering into bottom sheets.

Highlight games first.

Highlight communities second.

Highlight people third.

Discovery before filtering.

---



# Empty States

Never display empty black pages.

Every empty state should help users continue.

Examples:

- suggest communities
- recommend players
- recommend games
- explain the next action
- provide a clear CTA

Every empty state should feel alive.

---



# Forms

Forms should feel lightweight.

Avoid oversized containers.

Avoid unnecessary labels.

Prefer intelligent placeholders.

Group related fields.

Reduce cognitive load.

---



# Motion

Animations should feel like iOS.

Motion should be:

Fast.

Natural.

Subtle.

Purposeful.

Avoid decorative animation.

Use motion only when it improves understanding.

---



# Components

Every component should follow:

Less visual weight.

More clarity.

Rounded corners should feel modern but restrained.

Borders should be subtle.

Elevation should come from contrast rather than shadows.

---



# Information Density

Do not overload users.

Progressively reveal complexity.

Beginners should never feel intimidated.

Advanced users should still find powerful features.

---



# Mobile Experience

Everything should be optimized for thumb usage.

Primary actions should be easy to reach.

Bottom sheets are preferred over large modal windows.

Scrolling should feel intentional.

Avoid long endless forms.

---



# Arabic First

Nexus is designed primarily for Arabic-speaking gamers.

Arabic is not a translated version.

Arabic is a first-class language.

Every screen must be reviewed in:

- Arabic
- English

Mixed Arabic and English content must render correctly.

RTL must feel intentional.

Not mirrored.

---



# Accessibility

Accessibility is part of premium quality.

Support:

- Dynamic text
- VoiceOver
- Proper focus order
- High contrast
- Reduced motion
- Large touch targets

Accessibility should never be treated as optional.

---



# Performance

Performance is part of design.

Avoid heavy visual effects.

Avoid unnecessary rendering.

Avoid excessive blur.

Avoid expensive animations.

Interfaces should feel responsive on mid-range devices.

---



# Decision Framework

Before implementing any new screen ask:

Does this improve usability?

Does this improve clarity?

Does this improve perceived quality?

Does this reduce user effort?

Does this feel native?

Would Apple ship this interaction?

If the answer is no, redesign it.

# Cursor Rules

For every future implementation:

1. Review this document first.
2. Explain why the current UI violates these principles.
3. Design the simplest solution.
4. Prefer improving usability over adding visuals.
5. Maintain consistency with all previous screens.
6. Never redesign components independently without considering the whole product.
7. If unsure between beauty and simplicity, choose simplicity.
8. If unsure between branding and usability, choose usability.
9. Every completed screen should increase the feeling that Nexus is a premium native application.
10. Every implementation should move Nexus closer to becoming the highest-quality gaming application for Arabic-speaking users.  
  
# Design Authority
  This document is the single source of truth for the visual identity of Nexus.
  Do not introduce new design patterns, spacing systems, navigation styles, animations or visual components unless they are consistent with this document.
  When in doubt, preserve consistency over creativity.
